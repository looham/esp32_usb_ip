## CMD_UNLINK 处理协议

CMD_UNLINK要求每个请求必须对应一个RET_UNLINK：
- **传输未完成**：取消传输，发送RET_UNLINK(cmd_seqnum, -ECONNRESET)，不再发RET_SUBMIT
- **传输已完成但RET_SUBMIT未发**：发RET_UNLINK(cmd_seqnum, 实际状态码)，不再发RET_SUBMIT
- **RET_SUBMIT已发**：直接发RET_UNLINK(cmd_seqnum, 0)
- **传输已完全处理**：直接发RET_UNLINK(cmd_seqnum, 0)
- **顺序保证**：RET_SUBMIT必须在RET_UNLINK之前到达客户端

## 分块传输协议

### 触发条件
当 `transfer_buffer_length > CONFIG_USBIPD_MAX_CHUNK_SIZE`（默认4096字节）时启用分块。

### 分块传输流程
1. CMD_SUBMIT到达，判断需要分块
2. 从ChunkedPool分配ChunkedTransfer，从CallbackArgsPool分配esp32_callback_args
3. 注册到 `chunked_transfers_[seqnum]`
4. 同端点串行化：如端点已有活跃分块传输，入队 `deferred_urbs_` 等待
5. 提交第一个chunk的usb_transfer_t
6. chunk完成后回调：
   - IN：数据拷入PSRAM累积缓冲区，如未到短包/ZLP则提交下一chunk
   - OUT：递增pending_count，全部完成后发ret_submit
7. 全部chunk完成→发送RET_SUBMIT（IN从PSRAM缓冲区发送）→释放ChunkedTransfer回池

### IN分块传输
- 每个chunk完成后将数据拷入PSRAM `in_data` 缓冲区对应偏移
- DMA transfer可立即释放，ret_submit时从PSRAM缓冲区scatter-gather发送
- 设备返回短包/ZLP时置 `in_short=true`，CANCELED回调跳过重提交

### OUT分块传输
- 逐chunk提交，每个chunk完成后提交下一个
- 全部完成后统一发ret_submit

### 端点串行化
- `active_chunked_eps_`：记录当前有活跃分块传输的端点
- `deferred_urbs_`：同端点新传输入队等待
- 前一个分块传输完成后自动出队处理下一个

## ESP32 USB Host 接口

### USB Host 初始化
```
usb_host_config_t {
    .skip_phy_setup = false,
    .intr_flags = ESP_INTR_FLAG_LEVEL3,
    .enum_filter_cb = nullptr,
}
```

### USB Host 客户端
```
usb_host_client_config_t {
    .max_num_event_msg = 15,
    .async = {
        .client_event_callback = ...,
        .callback_arg = ...,
    }
}
```

### 设备事件
| 事件 | 说明 |
|:---|:---|
| USB_HOST_CLIENT_EVENT_NEW_DEV | 新设备插入 |
| USB_HOST_CLIENT_EVENT_DEV_GONE | 设备拔出 |

### 传输回调参数（esp32_callback_args）
| 字段 | 类型 | 说明 |
|:---|:---|:---|
| handler | Esp32DeviceHandler* | 处理器指针 |
| seqnum | uint32_t | USB/IP命令序列号 |
| transfer_type | usb_transfer_type_t | 传输类型 |
| is_out | bool | 方向 |
| chunked | ChunkedTransfer* | 分块传输状态（可空） |
| unlinked | bool | CMD_UNLINK已到达 |
| unlink_cmd_seqnum | uint32_t | UNLINK命令序列号 |
| original_transfer_buffer_length | uint32_t | 原始传输长度 |
| transfer | TransferHandle | 传输句柄（拥有所有权） |

### ChunkedTransfer 结构
| 字段 | 类型 | 说明 |
|:---|:---|:---|
| transfers | vector\<usb_transfer_t*\> | 有序分块transfer列表 |
| seqnum | uint32_t | USB/IP命令序列号 |
| is_out | bool | OUT方向标志 |
| transfer_buffer_length | size_t | 原始请求总长度 |
| worst_status | usb_transfer_status_t | 所有chunk中最差状态 |
| pending_count | atomic\<int\> | 未完成chunk计数 |
| total_actual_length | atomic\<size_t\> | 累计实际传输字节 |
| transfer_started | atomic\<bool\> | 首个chunk成功标志 |
| ep_address | uint8_t | 端点地址 |
| in_data | uint8_t* | PSRAM IN累积缓冲区 |
| in_data_size | size_t | IN缓冲区总大小 |
| in_short | atomic\<bool\> | IN短包/ZLP标志 |
| current_chunk | int | 下一个待提交chunk索引 |
| transfer_flags | uint32_t | 原始transfer_flags |

## 特殊控制请求（Tweak）

ESP32部分控制请求不提交transfer，直接由handler处理：

| 请求 | 判断条件 | 处理方式 |
|:---|:---|:---|
| CLEAR_HALT | request=CLEAR_FEATURE, recip=Endpoint, value=0 | 调用usb_host_endpoint_clear_halt |
| SET_INTERFACE | request=SET_INTERFACE, recip=Interface | 调用usb_host_interface_set |
| SET_CONFIGURATION | request=SET_CONFIGURATION, recip=Device | 调用usb_host_device_set_configuration |
| RESET_DEVICE | request=SET_FEATURE, type=Class+Other, value=Reset | 调用usb_host_device_reset |

返回值：-1=需要提交transfer，0=tweak成功，>0=esp错误码

## GPIO TCP 协议（端口8080）

行文本协议，`\n`结尾，单客户端，30秒超时。

| 命令 | 格式 | 响应 | 说明 |
|:---|:---|:---|:---|
| PING | `PING\n` | `PONG\n` | 心跳检测 |
| GET | `GET <pin>\n` | `<0\|1>\n` 或 `ERR\n` | 读取引脚电平 |
| SET | `SET <pin> <0\|1>\n` | `OK\n` 或 `ERR\n` | 设置输出电平 |
| LED | `LED <r> <g> <b>\n` | `OK\n` 或 `ERR\n` | 设置RGB LED颜色（0-255） |

- GET允许引脚：输出白名单(1,2,21,47) + 按钮列表(0,14,39)
- SET允许引脚：输出白名单(1,2,21,47)
- SET时自动配置为输出模式并上拉
