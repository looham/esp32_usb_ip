## 模块外部特性

### main.cpp（程序入口）
- `app_main()`: FreeRTOS入口，创建主线程（Core 1, prio 5, 8KB栈）
- `init()`: 初始化NVS→板级→WiFi→USB Host
- `main()`: 创建Esp32Server，监听端口3240，循环打印内存信息
- 依赖：asio、spdlog、nvs_flash、esp_pthread、lwip

### board.cpp / board.h（板级管理）
- `board_setup()`: LED初始化(WS2812/GPIO48)→WiFi状态回调→按钮任务→GPIO TCP服务
- `init_usb_host()`: 安装USB Host库，启动USB事件处理线程（Core 1, prio 10）
- `board_sync_wifi_led()`: WiFi状态同步到LED（红=断开，黄=连接中，绿=已连接）
- LED状态：启动红→WiFi连接中黄→已连接绿→单击蓝→长按切换→5秒长按紫(清WiFi)
- 按钮事件：
  - 单击BOOT：LED变蓝
  - 长按1秒：切换LED开关
  - 超长按5秒：清除WiFi配置，显示紫色，3秒后重启

### Esp32Server（ESP32 USB/IP 服务器适配层）
- 封装 `Server`（usbipdcpp核心），管理ESP32 USB Host设备
- `init_client()`: 注册USB Host客户端（最大15事件消息）
- `bind_host_device()`: 读取设备/配置描述符，claim接口，创建UsbDevice+Esp32DeviceHandler，添加到Server
- `unbind_host_device()`: 释放接口，从Server移除设备
- `start()`: 设置线程核心亲和性（NetworkIO/SessionMain/SessionSender均绑Core 1），启动Server+client事件线程
- `stop()`: 停止Server，释放所有接口，等待client事件线程结束
- 设备热插拔：
  - NEW_DEV事件→打开设备→读取描述符→绑定到Server
  - DEV_GONE事件→标记设备移除→通知handler→强制关闭Session→释放接口→关闭设备

### Esp32DeviceHandler（ESP32 设备处理器）
- 继承 `AbstDeviceHandler`，处理ESP32 USB Host的实际传输
- **分块传输（ChunkedTransfer）**：当transfer_buffer_length > chunk_size(默认4KB)时拆分
  - 每个chunk独立 `usb_transfer_t`，共享一个 `ChunkedTransfer` 状态
  - IN传输：PSRAM累积缓冲区，chunk完成后拷入，ret_submit时统一发送
  - OUT传输：逐chunk提交，全部完成后统一ret_submit
  - 端点串行化：同端点同时仅一个分块传输活跃，新传输入队等待
- **传输回调**：
  - `transfer_callback()`: 非分块传输完成回调
  - `chunked_transfer_callback()`: 分块传输单个chunk完成回调
- **特殊请求tweak**：SET_CONFIGURATION/SET_INTERFACE/CLEAR_HALT/RESET_DEVICE直接处理，不提交transfer
- **CMD_UNLINK处理**：根据传输状态（未完成/已完成/已发送RET_SUBMIT）分别处理
- **对象池**：`esp32_callback_args`（64个）、`ChunkedTransfer`（16个），避免频繁new/delete
- `custom_transfer_io = true`：使用分块读写（send_transfer_data/recv_transfer_data），避免分配大连续缓冲区

### Server（USB/IP 核心服务器）
- 管理设备列表：`available_devices`（可导入）/ `using_devices`（已导入）
- 网络IO线程：asio io_context，accept新连接
- Session管理：每个客户端连接创建一个Session，自动管理生命周期
- 线程回调：`before_thread_create_callback` / `after_thread_create_callback`（嵌入式平台设核心亲和性）
- `ServerNetworkConfig`：socket收发缓冲区(128KB)、TCP_NODELAY

### Session（客户端会话）
- 三线程模型：run_thread(主) + network_io + sender_thread
- 双缓冲发送队列：write_buffer(生产者) ↔ read_buffer(消费者)，交换时短暂加锁
- `submit_ret_submit()` / `submit_ret_unlink()`: 异步提交响应，线程安全
- `enqueue_ret_*()` + `wakeup_sender()`: 批量入队+统一唤醒

### AbstDeviceHandler（设备处理器抽象基类）
- `receive_urb()`: 处理CMD_SUBMIT，纯虚函数
- `handle_unlink_seqnum()`: 处理CMD_UNLINK，纯虚函数
- `on_new_connection()` / `on_disconnection()`: 连接/断开回调
- `is_device_removed()` / `on_device_removed()`: 设备热插拔通知
- `custom_transfer_io` 标志：true时走分块IO路径，false时走连续内存scatter-gather路径
- transfer_handle 接口：alloc/get_buffer/get_actual_length/get_offset/iso_descriptor/free

### board_utils 组件（C语言）

#### pin_define.h（引脚定义）
- LED_PIN = 48（WS2812，RMT驱动）
- BOOT_BUTTON_GPIO = 0（Strapping，低电平有效）
- BUTTON1_GPIO = 14, BUTTON2_GPIO = 39
- GPIO输出：GPIO_OUT_1(1), GPIO_OUT_2(2), GPIO_OUT_3(21), GPIO_OUT_4(47)
- 白名单机制：`pin_board_output_allowed()`, `pin_board_input_allowed()`, `pin_board_is_button()`
- 可选DVP摄像头：GPIO 4-18（与通用GPIO冲突，默认不启用）

#### led_manager（LED管理）
- `led_config_t`：gpio_num, led_num, brightness
- `led_init()`, `led_set_color()`, `led_set_all()`, `led_clear()`
- 特效：`led_breathing()`, `led_rainbow()`, `led_blink()`
- `led_set_brightness()`, `led_deinit()`

#### wifi_manager（WiFi管理）
- `wifi_init()`: NVS/netif/WiFi驱动初始化→NVS有配置则STA连接→失败则AP配网
- `wifi_status_t`：initialized/connected/ap_mode_active/ssid/rssi/ip/gw/netmask
- `wifi_set_status_callback()`: 状态回调（0=断开, 1=连接中, 2=已连接）
- AP配网门户：SSID "登录-192.168.4.1"，HTTP 80端口，POST /config保存WiFi配置
- `wifi_clear_saved_config()`: 清除NVS中的WiFi配置

#### gpio_tcp_server（GPIO TCP服务）
- 独立端口8080，纯LwIP socket，Core 0，低优先级
- 行文本协议（\n结尾）：`PING`, `GET <pin>`, `SET <pin> <0|1>`, `LED <r> <g> <b>`
- 单客户端，30秒超时

#### button_manager（按钮管理）
- `button_id_t`：BUTTON_BOOT, BUTTON_1, BUTTON_2
- `button_init()`, `button_get_state()`, `button_clear_state()`

### usbipdcpp 核心库数据结构

#### UsbDevice（USB设备描述）
- path, busid, bus_num, dev_num, speed, vendor_id, product_id
- device_bcd(Version), device_class/subclass/protocol
- configuration_value, num_configurations
- interfaces列表, ep0_in/ep0_out
- handler (shared_ptr<AbstDeviceHandler>)
- `with_handler<T>()`: 创建并绑定handler

#### UsbInterface（USB接口描述）
- interface_class/subclass/protocol
- endpoints列表
- handler (shared_ptr<VirtualInterfaceHandler>)

#### UsbEndpoint（USB端点描述）
- address, attributes, max_packet_size, interval
- direction(): In/Out, is_ep0()

#### SetupPacket（USB Setup包）
- request_type, request, value, index, length
- 命令判断：is_clear_halt_cmd(), is_set_interface_cmd(), is_set_configuration_cmd(), is_reset_device_cmd()

#### ChunkedTransfer（分块传输状态）
- transfers: 有序usb_transfer_t列表
- seqnum, is_out, transfer_buffer_length
- worst_status, pending_count, total_actual_length
- in_data: PSRAM IN累积缓冲区
- in_short: 短包/ZLP标志
- current_chunk: 下一个待提交索引

## 编译配置

### ESP32-S3 配置
- Flash: QIO 16MB, CPU 240MHz, Octal PSRAM
- PSRAM: WiFi/LwIP放PSRAM，保留192KB内部RAM给USB DMA
- USB Host: Hub支持 + IN bias
- TCP: 最大16 socket, 8活跃TCP, 窗口11400

### ESP32-P4 配置
- Flash: QIO 32MB, PSRAM 80MHz
- USB Host: Hub支持, DWC DMA在PSRAM
- 以太网: OPENETH
- 依赖: esp_wifi_remote + esp_hosted（WiFi通过外部芯片）

### 分区表
| 名称 | 类型 | 子类型 | 偏移 | 大小 |
|:---|:---|:---|:---|:---|
| nvs | data | nvs | 0x9000 | 24KB |
| phy_init | data | phy | 0xf000 | 4KB |
| factory | app | factory | 0x10000 | 4MB |
| storage | data | - | - | 256KB |

### Kconfig 菜单
- **WiFi配置**: USBIPD_WIFI_SSID, USBIPD_WIFI_PASSWORD
- **分块传输**: USBIPD_ENABLE_CHUNKED_TRANSFER(默认y), USBIPD_MAX_CHUNK_SIZE(默认4096, 范围64-16384)
